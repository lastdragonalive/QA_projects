### Часть III: Тестирование API

#### **Шаги реализации**: 
- изучить требования к бэкенду приложения
- ознакомиться с документацией к API в Apidoc
- составить чек-лист проверок, покрывающих часть требований (см. Объекты тестирования)
- провести тестирование API с помощью Postman
- оформить баг-репорты

#### **Объекты тестирования**:
1) Работа с курьерами:
   * POST /api/v1/courier
   * DELETE /api/v1/courier/:id
2) Работа с заказами:
   * GET /api/v1/orders/track
3) Работа с БД:
   * проверка появления или удаления введенных данных при создании или удалении учетной записи курьера
  
#### Итоги тестирования:
   С **тестовой документацией** можно ознакомиться [*здесь*](https://docs.google.com/spreadsheets/d/1y7da3ir0CxBrZnm9IGOaU0Vwa3_xu97pGPv-5Ph4P5U/edit?usp=sharing)\
   С **баг-репортами** можно ознакомиться [*здесь*](https://docs.google.com/spreadsheets/d/1p22h-UMbukmyQkqVs7ZA3UdtP4yRNSJJjFq-jofiVYM/edit?usp=sharing)\
  Кроссбраузерное тестирование верстки и логики экрана "Статус заказа" осуществлялось по чек-листу из 100 проверок. Среди них 16 имеют статус "FAILED", 4 бага являются критическими.   Валидация полей экрана "Сделать заказ" была протестирована на основе данных, полученных с помощью разбиения на классы эквивалентности и выделения граничных значений. Было найдено 19 багов. Исследовательское тестирование прочих элементов веб-приложения вне вышеописанной тестовой документации выявило 10 багов.\
  Тестирование мобильного приложения для курьеров Яндекс.Самокат (версия 2.0) было выполнено на физическом устройстве Samsung Galaxy S23 Ultra (Android 14) с разрешением экрана 3088x1440 и диагональю 6,8 дюйма. 8 из 21 проверки не были пройдены, пять багов - критические.\
  При составлении чек-листа проверок для API Яндекс.Самокат (1.0.0.) также применялась техника КЭ и ГЗ. Тестирование осуществлялось с помощью Postman, а также Cygwin для взаимодействия с БД сервиса. В чек-лист вошло 52 проверки, 30 из которых - со статусом "FAILED" (включая 4 критических бага).\
  В результате проведенного тестирования в сумме было обнаружено 83 бага, 38% которых имеют приоритет "Критический". Наша команда рекомендует исправить найденные ошибки и передать приложение на повторное тестирование.

## **Тестирование API Яндекс.Прилавок**
**Поставленная задача**: протестировать новую функциональность бэкенда сервиса Яндекс.Прилавок

#### **Шаги реализации**: 
- изучить документацию к API в Apidoc  
- проанализировать требования к новой функциональности бэкенда Яндекс.Прилавка
- составить чик-лист проверок, покрывающих только новую функциональность (выделена жирным шрифтом в документации)
- провести тестирование API с помощью Postman
- оформить баг-репорты 
- составить отчет о проведенном тестировании и его результатах

#### **Объекты тестирования**:
1) Новая функциональность в работе с наборами:
   * POST /api/v1/kits/{id}/products — добавить продукты в набор
2) Новая функциональность в работе с курьерами:
   * POST /fast-delivery/v3.1.1/calculate-delivery.xml — возможность доставки и её стоимость курьерской службой «Привезём быстро»
3) Новая функциональность в работе с корзиной:
   * GET /api/v1/orders/:id — получить список продуктов в корзине
   * PUT /api/v1/orders/:id — добавить продукты в корзину
   * DELETE/api/v1/orders/:id — удалить корзину

## Тестирование мобильного приложения Яндекс.Метро
**Поставленные задачи**:
1) провести функциональное тестирование по частям приложения, затронутых рефракторингом
2) провести регрессионное тестирование

**Объекты тестирования**:
1) Функциональности, затронутые рефракторингом:
   * Методы построения маршрута
   * Метод выбор станции на карте и отображение карточки станции
   * Порядок сохранения станций в истории поиска
   * Логика изменения времени маршрута
   * Поведение интерфейса при смене ориентации экрана
   * Логика работы лонг-тапа по станции метро
   * Сообщение об ошибке при отсутствии интернет-соединения
2) Вся оставшаяся функциональность приложения

**Шаги реализации**:
- изучить требования к мобильному приложению
- составить чек-лист проверок, покрывающих изменённые функциональности:  
- составить чек-лист для проведения регрессионного тестирования
- протестировать приложение на устройстве-эмуляторе в Android Studio по чек-листам
- оформить баг-репорты
- написать отчёт о тестировании

**Тестовое окружение**:
  * Устройство: Эмулятор Android Studio (AVD)
  * Модель: HUAWEI Honor 8
  * ОС: Android 9.0 Pie (API 28)
  * Разрешение экрана: 1080 × 1920
  * Диагональ: 5.5 дюйма
  * Версия приложения: Яндекс Метро 3.6
